# Sample Prompts by Section

## Shallow Knowledge + Planning (Understand)

Understand the space before deciding.

* “Deconstruct this topic into 5 key ideas I need to **understand first**, sorted from most foundational to most advanced.”
* “Extract 10 unknowns and turn them into planning questions, sorted by which answers would **most change the direction of the work**.”
* “Abstract this messy brief into 3 possible core decisions I may actually be making, sorted by **likely impact on scope, risk, and effort**.”
* “Identify 7 assumptions I may be making without realizing it, sorted by which ones would **most affect the outcome if wrong**.”
* “List 5 constraints I should clarify before moving forward, sorted by which constraints would **most narrow the solution space**.”
* “Break this idea into 4 plausible approaches, sorted by which approach would **best fit the stated goals and limits**.”

Understand the terrain before acting and order the response to make the next-step decisions more obvious.

## Deep Knowledge + Planning (Refine)

Refine direction, tradeoffs, and constraints.

* “Annotate this proposal with 7 **hidden assumptions, tradeoffs, and risks**, sorted by which ones would most affect the quality of the final decision.”
* “Filter this design brief down to the 5 **decisions that matter most**, sorted by which decisions would have the biggest downstream impact.”
* “Hypothesize 4 viable approaches and explain **when each is best**, sorted by best overall fit for the stated goals and constraints.”
* “Compare these 3 options and identify the **tradeoff that should drive the decision**, sorted by which option creates the clearest advantage once that tradeoff is prioritized.”
* “Tighten this plan by identifying 6 **constraints that are still too vague**, sorted by which unclear constraints create the most execution risk.”
* “Refine this strategy into 4 sharper versions that make the **priority decisions explicit**, sorted from most conservative to most aggressive.”

Refine judgment, reduce ambiguity, and clarify what matters most.

## Shallow Knowledge + Create (Verify)

Produce something usable without full mastery, while exposing the rationale, assumptions, and checks needed to validate it.

* “Turn this rough idea into 3 usable drafts, sorted from safest and most conventional to most ambitious, and **explain why each is valid**.”
* “Extract the minimum structure this output needs to be effective, sorted by **what matters most to get right first**, and explain the reasoning behind that order.”
* “Rewrite this into a practical first version I can use now, while highlighting the 5 choices I may still need to make and **what I should verify before using it**.”
* “Draft this for me in a usable form, then list the **assumptions, risks, and checks** I should review before relying on it.”
* “Create a first version that follows the **most defensible pattern**, and explain what makes that pattern appropriate here.”
* “Write this so I can use it immediately, then add a short note explaining **what could still be wrong or incomplete**.”

Produce something usable now while making the reasoning, assumptions, and remaining risks visible.

## Deep Knowledge + Create (Create)

Produce the final form.

A useful checkpoint prompt at this stage is: “List the assumptions you are making that are not covered by the agreed requirements, ordered from highest to lowest impact.” In practice, this prompt can be used at any time to help reveal which quadrant the operator is actually in before proceeding.

Once all assumptions between the operator and the LLM have been aligned, proceed with the production of the final form.

* “Write this to match the **agreed structure, tone, and constraints**.”
* “Implement this as specified, with **no variation beyond the stated requirements**.”
* “Turn this into the **final version that should ship**.”
* “Produce the finished version using the **approved format and wording style**.”
* “Complete this exactly as defined, preserving the **agreed logic, structure, and boundaries**.”
* “Deliver the final output in its **ready-to-use form**, with no exploratory alternatives.”

Produce the final form through faithful execution rather than exploration or variation.
