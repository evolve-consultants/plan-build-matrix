# Draft Abstract: [Your Design System Name]

**Building a Design System That Actually Shipped**

We didn't start with a grand vision. We started with chaos—inconsistent components, duplicated work, and designers and developers speaking different languages. This talk walks through how we built a design system that solved real problems instead of creating new ones.

You'll learn:
- **Why we started** (the breaking point that forced our hand)
- **What we got wrong first** (and how we fixed it)
- **How we kept it alive** (governance, adoption, the stuff that's harder than the design)
- **What's actually working now** (and the metrics that proved it)

Whether you're starting from scratch or rescuing a system that's gathering dust, this is a practical look at the tradeoffs, the wins, and the unsolved problems we're still wrestling with.
