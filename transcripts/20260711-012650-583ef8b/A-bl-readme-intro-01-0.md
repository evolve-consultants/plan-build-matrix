# Friendly README Intro Template

Here's a structure you can adapt:

---

## 🎯 What is [Project Name]?

[One sentence: what it does for users]

Need to [solve X problem]? This project helps you [accomplish Y] without [pain point Z].

## ✨ Why use it?

- **[Benefit 1]** — [one short sentence]
- **[Benefit 2]** — [one short sentence]
- **[Benefit 3]** — [one short sentence]

## 🚀 Quick start

```
[One minimal example: 3-5 lines max]
```

That's it! See [docs/guide] for more.

---

## Tips for keeping it friendly:

- **Lead with the "why"** not the "what" — why would someone care?
- **Use contrasts** — "no more X, just Y" works well
- **Be specific** — "faster deployment" beats "better performance"
- **One example beats three paragraphs** of explanation
- **Emoji + headers** break up visual monotony (sparingly)
- **Keep sentences short** — aim for one idea per line
