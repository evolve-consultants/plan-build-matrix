### Recommendation
**Start with Option A (diagnostic)** because you're operating on vague feedback. This doesn't cost much and immediately tells you whether to do Option B (small fixes) or invest in Option C (bigger redesign).

**How to run a quick diagnostic:**
1. Collect existing feedback: support tickets, analytics heatmaps, session recordings (if available)
2. Do 3–5 short user interviews: "Show me how you use the dashboard. Where did you get stuck?"
3. Synthesize: cluster complaints into themes (navigation, metrics, visual hierarchy, jargon, performance)
4. Prioritize: which problems block the most users or block them the hardest?
